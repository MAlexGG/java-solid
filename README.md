# :thought_balloon: SOLID Principles

Repositorio educativo para aprender y comprender los principios SOLID aplicados a Java mediante ejemplos prácticos.


 ## Descripción
Este proyecto tiene como objetivo explicar los cinco principios SOLID, un conjunto de buenas prácticas de diseño orientado a objetos que permiten desarrollar aplicaciones más mantenibles, escalables y fáciles de evolucionar.
A través de ejemplos sencillos, se muestran problemas habituales de diseño y cómo resolverlos aplicando estos principios.


## Objetivos
- Comprender qué son los principios SOLID.
- Aprender a identificar problemas de diseño en una aplicación.
- Aplicar buenas prácticas de programación orientada a objetos.
- Reducir el acoplamiento entre componentes.
- Mejorar la mantenibilidad y la escalabilidad del código.


## ¿Qué es SOLID?
SOLID es un acrónimo formado por cinco principios fundamentales del diseño de software:

### S - Single Responsibility Principle (SRP)
Una clase debe tener una única responsabilidad y un único motivo para cambiar.

### O - Open/Closed Principle (OCP)
Las entidades de software deben estar abiertas a la extensión, pero cerradas a la modificación.

### L - Liskov Substitution Principle (LSP)
Las clases derivadas deben poder sustituir a sus clases base sin alterar el comportamiento esperado del sistema.

### I - Interface Segregation Principle (ISP)
Es preferible utilizar interfaces pequeñas y específicas antes que una interfaz grande y genérica.

### D - Dependency Inversion Principle (DIP)
Los módulos de alto nivel y los módulos de bajo nivel deben depender de abstracciones y no de implementaciones concretas.
